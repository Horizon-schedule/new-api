# 消费日志排查指南

面向管理员与客户：当用户反馈「未使用 API 却出现大量消费日志」时，可按本清单逐步排查。

## 日志字段含义

| 字段 | 含义 |
|------|------|
| **时间（列表列）** | 请求**结束**并写入扣费记录的时间（`created_at`） |
| **耗时（绿色）** | 从网关开始转发到请求完成的总耗时（`use_time`，秒） |
| **首 token（橙色，流式）** | 首 token 延迟 FRT（毫秒，存在 `other.frt` 时） |
| **请求开始时间（详情）** | 推算值：`created_at - use_time` |
| **客户端 IP** | 仅当用户在设置中开启「记录 IP」时才会写入 |

相邻两条日志的「时间」差值**不等于**上一条请求的上行耗时；并发请求会以各自结束时间排序。

## 消费日志从何而来

以下情况**不会**产生用户令牌的消费日志：

- 管理员在渠道管理中的「模型测试」（固定为系统用户）
- 操练场（Playground）调试（令牌名通常为 `playground-xxx`）

以下情况**会**产生消费日志：

- 外部客户端携带用户 API Key 调用 `/v1/chat/completions` 等接口
- 成功预扣/结算且 `LogConsumeEnabled` 开启

因此，出现带真实令牌名、较大 prompt token、流式标记的消费记录，通常表示 **Key 已被外部程序调用**。

## 管理员排查步骤

1. **打开日志详情 →「请求溯源」**
   - 查看本条请求的结束时间、推算开始时间、客户端 IP
   - 查看该令牌的**首次消费时间**、**最近消费时间**、**历史 IP 列表**
2. **按令牌名筛选**
   - 在日志页用 `token_name` 过滤，确认是否集中在某一把 Key
3. **核对 IP**
   - 若 IP 为空：请让用户在「个人设置」中开启「记录 IP」，便于后续追踪
   - 若有多个不同 IP：Key 可能已配置在多台设备/服务（Cursor、OpenRouter、CI、脚本等）
4. **检查令牌状态**
   - 在令牌管理中查看该 Key 是否仍启用、是否设置了 IP 白名单
   - 若来源不明：**立即禁用或轮换**该令牌
5. **排除误报场景**
   - 确认不是同事/集成服务在使用同一账号下的 Key
   - 确认不是将 Key 填入了公开仓库或浏览器插件

## 给客户的说明模板

> 系统仅在 API Key 被外部程序真实调用时才会扣费并写入消费日志。请在日志详情中查看「请求溯源」中的 IP 与令牌首次消费时间。若您未主动调用，请检查 Key 是否曾配置在 Cursor、OpenRouter、自动化脚本或第三方工具中；建议在令牌管理中禁用旧 Key 并创建新 Key，同时在设置中开启「记录 IP」以便后续审计。

## 相关 API（管理员）

```
GET /api/log/token_audit?user_id={uid}&token_id={tid}
GET /api/log/token_audit?user_id={uid}&token_name={name}
```

返回：首次/末次消费时间、消费条数、历史 distinct IP、令牌最近访问时间等。

用户自助：

```
GET /api/log/self/token_audit?token_id={tid}
```
