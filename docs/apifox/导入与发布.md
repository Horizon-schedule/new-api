# Apifox 一键导入与发布（给客户看的在线文档）

本目录已生成可直接导入 Apifox 的全部文件，**无需手写接口**。

## 文件说明

| 文件 | 用途 |
|------|------|
| `openapi.json` | 接口定义（GPT / Qwen / DeepSeek / Claude / Gemini） |
| `environments/默认环境.json` | 环境变量（网关地址 + API Key） |
| `markdown/*.md` | 6 篇客户可见的说明文档 |

---

## 第一步：只需改 2 个地方（必做）

打开 `environments/默认环境.json`，修改：

1. **`baseUrl`** → 你的网关地址，例如 `https://api.example.com`（**不要用** `docs.newapi.pro`）
2. **`api_key`** → 控制台创建的测试令牌 `sk-xxx`（用于 Apifox 内调试，发布给客户后客户用自己的 Key）

---

## 第二步：导入 Apifox（约 5 分钟）

### 2.1 新建项目

1. 打开 [Apifox](https://apifox.com) → **新建项目** → 名称如「AI 模型 API」

### 2.2 导入接口

1. 点左上角 **项目设置**（齿轮）→ **导入数据** → **手动导入** → **上传文件**
2. 类型选 **OpenAPI/Swagger** → 上传 **`openapi.json`**
3. 导入到：**根目录**；冲突：**覆盖**；确认导入
4. 左侧 **接口管理** 应出现约 **7 个接口**

> 若左侧仍为空，见 **[导入失败排查.md](./导入失败排查.md)**，改用 **`postman-collection.json`**（Postman 格式）导入，最稳定。

> ⚠️ 请用 `docs/apifox/openapi.json`，不要用 `docs/openapi/relay.json`（全量 7000 行，易导入失败）。

### 2.3 导入环境

1. 右上角 **环境管理** → **导入**
2. 选择 **`environments/默认环境.json`**
3. 勾选 **默认环境** 为当前使用环境

### 2.4 配置鉴权（全局）

1. 项目 **设置** → **Auth**（或根目录认证）
2. 类型：**Bearer Token**
3. Token 填：`{{api_key}}`

### 2.5 导入 Markdown 文档（给客户看的说明页）

1. 左侧 **文档** → **新建文档** → 类型 **Markdown**
2. 按顺序创建 6 篇，标题与文件对应，内容 **整篇复制粘贴**：

| 文档标题 | 复制自文件 |
|----------|------------|
| 引言与快速开始 | `markdown/00-引言与快速开始.md` |
| GPT 系列 | `markdown/01-GPT系列.md` |
| Qwen 系列 | `markdown/02-Qwen系列.md` |
| DeepSeek 系列 | `markdown/03-DeepSeek系列.md` |
| Claude 与 Claude Code | `markdown/04-Claude与Claude-Code.md` |
| Gemini 系列 | `markdown/05-Gemini系列.md` |
| 模型列表与对照表 | `markdown/06-模型列表与对照表.md` |

3. 将 **「引言与快速开始」** 设为文档站 **首页**（文档设置 → 默认文档）

### 2.6 试跑一条接口

1. 打开 **Claude 聊天** 或 **创建聊天对话**
2. 右上角环境选 **默认环境**
3. 点击 **发送** → 应返回 200（若 401 检查 api_key；若 fetch failed 检查 baseUrl）

---

## 第三步：发布在线文档（客户访问链接）

1. 右上角 **分享** → **发布文档站** / **在线分享**
2. 设置 **自定义子域名**（如 `yourapi.apifox.cn`）或使用系统分配域名
3. 发布完成后复制链接，例如：`https://yourapi.apifox.cn`
4. 在你 new-api 后台 **系统设置 → API 信息** 中，把该链接填给客户

客户打开链接即可阅读 Markdown 说明，并在文档内 **在线调试**（需自行填写自己的 API Key）。

---

## 常见问题

| 现象 | 处理 |
|------|------|
| `fetch failed` / 500 | baseUrl 填成了文档站；改为你自己的网关 |
| 401 | api_key 为空或无效 |
| Claude 400 | 缺少请求头 `anthropic-version: 2023-06-01` |
| 模型不存在 | 控制台未启用该模型；用 `GET /v1/models` 查看可用列表 |

---

## 更新文档

修改 `openapi.json` 或 markdown 后，在 Apifox 中重新导入 OpenAPI / 粘贴文档，再 **重新发布** 文档站即可。
