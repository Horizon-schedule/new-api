# Apifox 导入后「什么都没有」？按这个做

## 原因说明

之前 `openapi.json` 里 `servers.url` 写成了 `{{baseUrl}}`，**不符合 OpenAPI 规范**，Apifox 会解析失败，左侧接口树就是空的。

已修复为合法地址 `https://api.example.com`。**请重新导入**（见下方）。

---

## 方法一：重新导入 OpenAPI（推荐）

1. 打开 Apifox 项目 **relay**
2. 点左上角 **项目设置**（齿轮图标），不是左侧「接口管理」里的新建
3. 左侧选 **导入数据**
4. 选 **手动导入** → **上传文件**
5. 文件类型选 **OpenAPI/Swagger**
6. 选择本目录文件：**`openapi.json`**（完整路径：`docs/apifox/openapi.json`）
7. 导入设置：
   - **导入到**：根目录 / 默认模块
   - **冲突处理**：**覆盖**（或「智能合并」）
   - 可勾选 **导入 Servers 为环境**
8. 点 **确认导入**
9. 回到左侧 **接口管理**，展开 **默认模块 → 接口**，应看到约 **7 个接口**

> ⚠️ 不要导入 `docs/openapi/relay.json`（7000+ 行全量文件），除非你知道如何处理。客户文档用 **`docs/apifox/openapi.json`** 即可。

---

## 方法二：若 OpenAPI 仍为空 → 用 Postman 格式（最稳）

Apifox 对 Postman Collection 导入兼容性更好：

1. **项目设置** → **导入数据** → **上传文件**
2. 类型选 **Postman**
3. 选择：**`postman-collection.json`**
4. 确认导入

导入后左侧会出现 **7 个文件夹**（模型列表、GPT、Qwen、DeepSeek、Claude、Gemini、Embeddings），共 **10 个请求**。

---

## 导入成功后：改你的网关地址

1. 右上角 **环境** → **管理环境**
2. 编辑 **开发环境** 或 **默认环境**
3. **前置 URL** 改为你的真实网关，例如：
   - 本地：`http://127.0.0.1:3000`
   - 线上：`https://api.你的域名.com`
4. 变量 **`api_key`** = 控制台 `sk-xxx`

（Postman 导入的改 Collection 变量 `baseUrl` 和 `api_key`）

---

## 导入 Markdown 文档（给客户看）

接口导入成功后，左侧切换到 **「文档」**（不是「接口管理」）：

1. **新建文档** → 类型 **Markdown**
2. 从 `markdown/` 文件夹复制 7 篇内容粘贴
3. **分享 → 发布文档站** 得到客户访问链接

---

## 自检清单

| 检查项 | 正确状态 |
|--------|----------|
| 导入文件 | `docs/apifox/openapi.json` 或 `postman-collection.json` |
| 左侧接口树 | 至少有 7 个 HTTP 接口或 10 个请求 |
| 前置 URL | 不是你的文档站 `docs.newapi.pro` |
| api_key | 已填完整 `sk-...` |

仍为空时：删除当前空项目，**新建项目**再按 **方法二 Postman** 导入一次。
